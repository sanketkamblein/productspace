Reference Objects
in Private Cloud

End-to-end UX design of three SAP Fiori applications enabling Deutsche Bahn to manage Reference Equipment and Reference Structures — reusable technical object templates for Enterprise Asset Management in SAP Private Cloud.

FPS1 · Fiori Launchpad
Background

The Challenge

Deutsche Bahn required a scalable way to define and reuse technical object templates across its vast asset portfolio. Rather than creating individual equipment records from scratch, Reference Objects act as master templates — defining shared structures, classifications, and attributes that can be instantiated across the network. The UX challenge was making this powerful but complex data model feel approachable in SAP Fiori Elements.

🏗️

Reference Equipment

Template-like equipment entities that define reusable characteristics, classes, partners, and measuring points — used as blueprints for real technical objects across the asset network.

🌳

Reference Structures

Hierarchical structural templates that can be reused across multiple plant locations. Includes Consistency Check workflows and Bill of Material views for configuration validation.

📋

Manage Structures

Administrative view for overseeing and managing Reference Structures across the system — with import/export capabilities and downstream consistency verification.

My Work

Scope & Contributions

This was a delivery-focused engagement — requirements came pre-defined from Deutsche Bahn. My ownership was in translating them into precise, developer-ready Fiori designs.

FPS1 Deliverables

  • Designed end-to-end UX for Manage Reference Equipment (List Report + Object Page)
  • Built Fiori Launchpad entry point with tile configuration
  • Designed all Object Page tabs: General Info, Structure, Classes, Characteristics, Partners, BOM, Reference Measuring Points, Where Used
  • Delivered RDS Library edit screens aligned with ISO 81346
  • Created full annotation-ready Figma handoff for development team

FPS2 Deliverables

  • Extended filter bar with advanced filter panel design
  • Introduced Floating Footer pattern for save/discard in edit mode
  • Refined RDS Library edit flow with additional detail states
  • Designed Manage Reference Structure: Import File, Consistency Check, BOM views
  • Developer coordination for annotation accuracy and responsiveness patterns
  • Stakeholder validation and documentation for final handoff

Design Process

  • Worked strictly within SAP Fiori Elements guidelines and private cloud constraints
  • Collaborated closely with developers to validate annotation feasibility
  • Iterated on edge cases: empty states, error handling, multi-language support
  • Reviewed implementation against design specs during sprint deliveries

Scope & Context

  • Customer-driven project — no independent discovery phase
  • Requirements sourced directly from Deutsche Bahn's asset management team
  • Three applications delivered: Manage Reference Equipment, Manage Reference Structure, Manage Structures
  • Strengthened system-wide reusability and configuration consistency in EAM
