Slide 3 — Vision and Goals

Efficient Buying

Our strategic priority: build an AI Agent that recommends the next best step in a multi-round sourcing auction — backed by pricing history, market data, and supplier behaviour patterns.

Core Vision

Recommend Next Step in Multi-Round Auction

An AI Agent monitors live sourcing events and recommends whether to open a new round, accept a bid, or push for a lower BAFO — powered by pricing history and event profiles.

AI Agent Pricing History Multi-Round Auction
Strategic Priority

Negotiation Support

AI analyses negotiation patterns and provides recommendations on counteroffers, concessions, and potential trade-offs. It can predict the likelihood of acceptance for a particular negotiation strategy or offer.

Counteroffer Analysis Trade-off Modelling
Strategic Priority

Auction Design Intelligence

Learning from past auctions to improve future processes. AI recommends the most suitable auction format based on the procurement scenario, market conditions, and historical data — including starting price, price increments, and duration.

Auction Format Historical Learning Parameter Optimisation
Pricing History — Pain Point

Eliminating Manual Supplier Qualification

Manually identifying and qualifying potential suppliers is time-consuming and prone to human error, leading to missed opportunities. The AI layer removes this bottleneck entirely.

Slide 4 — User Journey

Sourcing Process for Guided Buying

The current user journey for a multi-round auction — from sourcing request to deal close. This is the workflow the AI agent is built around.

Current Scenario — Task Flow

Sourcing requests with multiple suppliers flow through a manual series of decisions — compare, strategise, act, and close — with no AI assistance at any step.

Sourcing Request

Suppliers requested for sourcing event

→

Compare

View and compare the price from all bids

→

Strategise

Strategise to spend within given budget

→

Multiple Actions

Make multiple actions to lower the price; open second round

→

Lock & Close

Freeze, select suppliers, and close the deal

Where Agentic AI Steps In

1

Select Suppliers

AI recommends which suppliers to invite based on historical performance, item match, and compliance record.

2

Compare Sourcing Bids

Automated multi-attribute scoring of bids received — price, quality, delivery, compliance — removing manual bias.

3

Analyse & Decide

Analysis: benchmark vs. market price. Decision: open a new round, accept the bid, or request a revision.

4

Select Items & Quote Again

If a new round is warranted, the agent pre-fills recommended parameters for the next round based on BAFO patterns.

Slide 5 — Prioritised Requirements

Problem Statement & Requirements

Four requirements identified from the sourcing process analysis — the first three formed the MVP scope.

Requirement 1 — MVP

Assistance for Multi-Round Auction

"Lack of data-driven insights makes negotiations inefficient, leading to suboptimal agreements that fail to maximise value."
  • Archive old pricing data
  • Assess Supplier Bids & BAFO Analysis
  • Evaluate Market Conditions
  • Review Event Metrics
  • Recommend Next Move
Requirement 2 — MVP

Pricing History

"Learning from past auctions to improve future processes is often haphazard, leading to stagnant performance and lost opportunities for optimisation."
  • Identify non-codified items
  • Access past pricing for the item
  • Evaluate current and past market trend
  • Consolidate priority for the supplier's bid
Requirement 3 — MVP

Comparison

"Comparing complex bids across multiple criteria is labour-intensive and can result in inconsistent evaluations, impacting decision quality."
  • Analyse suppliers for the same event
  • Strategy of the auction
Requirement 5 — Post-MVP

Supplier's Review & Risk

"Identifying and mitigating risks associated with suppliers is challenging without comprehensive, real-time data analysis."
  • Real-time supplier risk signals
  • Geopolitical and financial exposure
  • Fraud and anomaly detection
Slide 6 — Architecture

MVP / POC Architecture

Built entirely on SAP BTP — Ariba Open API feeds real-time and historical bid data into SAP HANA DB, where AI Core and AI PAL run the grading and comparison models.

Tech Stack
SAP Ariba Open API SAP HANA DB SAP AI Core SAP AI PAL SAP AI Launchpad — HANA Model SAP BTP

Data Flow

📡

Ariba Open API

Pulls real-time bid data, market demand, commodity trends, and all supplier-provided details from live sourcing events.

🗄️

SAP HANA DB

Stores baseline price, history data, and bidding event records for similar objects and suppliers. Enables fast historical lookups.

🤖

AI Core / AI PAL

Runs the grading and analysis model. Scores supplier bids, detects BAFO patterns, and generates the Reward / Bargain recommendation with reasoning.

📊

Comparison Layer

Analyses multiple suppliers for the same event. Surfaces side-by-side scoring with profit, market trend, and baseline price context.

Architecture Design Decision

TASK-2: Build an Analysis model based on the Instruction Document provided in the sourcing events — using bidding event history for similar objects and suppliers to generate intelligent, context-aware recommendations directly within the Ariba workflow.
